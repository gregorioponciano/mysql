Roadmap MySQL

Este site é um guia para aprender **MySQL** e o uso do **MySQL Workbench**. Ele cobre desde os conceitos básicos até relacionamentos e comandos importantes.

Banco de Dados

Um banco de dados é um contêiner que armazena tabelas e outros objetos.

CREATE DATABASE loja;
USE loja;

Tabelas

Tabelas armazenam dados em linhas e colunas. Exemplo:

CREATE TABLE clientes (
    id INT AUTO_INCREMENT PRIMARY KEY,
    nome VARCHAR(100),
    email VARCHAR(100),
    telefone VARCHAR(15)
);

Relacionamentos

Relacionamentos ligam tabelas entre si usando **chaves estrangeiras**.

CREATE TABLE pedidos (
    id INT AUTO_INCREMENT PRIMARY KEY,
    cliente_id INT,
    data DATE,
    FOREIGN KEY (cliente_id) REFERENCES clientes(id)
);

MySQL Workbench

O Workbench é a ferramenta oficial da Oracle para administrar o MySQL. Com ele é possível:

  • Criar e modelar bancos de dados (modelagem ER)
  • Executar consultas SQL
  • Gerenciar usuários e permissões
  • Exportar e importar dados

Principais Comandos SQL

Inserir dados

INSERT INTO clientes (nome, email, telefone)
VALUES ('João Silva', 'joao@email.com', '11999999999');

Consultar dados

SELECT * FROM clientes;
SELECT nome, email FROM clientes WHERE id = 1;

Atualizar dados

UPDATE clientes SET email = 'novo@email.com'
WHERE id = 1;

Excluir dados

DELETE FROM clientes WHERE id = 1;

Esquema de Banco de Dados Completo

Para um sistema de e-commerce, precisamos de um esquema que ligue **produtos**, **clientes** e **pedidos**. A seguir, o código para criar todas as tabelas e suas relações. 


-- Use o banco de dados 'loja' criado anteriormente
USE loja;

-- 1. Tabela de Categorias
-- Organiza os produtos em grupos.
CREATE TABLE categorias (
    id_categoria INT PRIMARY KEY AUTO_INCREMENT,
    nome_categoria VARCHAR(50) NOT NULL
);

-- 2. Tabela de Produtos
-- Detalhes de cada produto. O id_categoria é a chave estrangeira.
CREATE TABLE produtos (
    id_produto INT PRIMARY KEY AUTO_INCREMENT,
    nome_produto VARCHAR(100) NOT NULL,
    descricao TEXT,
    preco DECIMAL(10, 2) NOT NULL,
    estoque INT NOT NULL DEFAULT 0,
    id_categoria INT,
    FOREIGN KEY (id_categoria) REFERENCES categorias(id_categoria)
);

-- 3. Tabela de Clientes
-- Informações dos clientes.
CREATE TABLE clientes (
    id_cliente INT PRIMARY KEY AUTO_INCREMENT,
    nome_cliente VARCHAR(100) NOT NULL,
    email VARCHAR(100) UNIQUE NOT NULL,
    endereco VARCHAR(255)
);

-- 4. Tabela de Pedidos
-- Registra cada compra. O id_cliente é a chave estrangeira.
CREATE TABLE pedidos (
    id_pedido INT PRIMARY KEY AUTO_INCREMENT,
    id_cliente INT,
    data_pedido DATETIME NOT NULL,
    status VARCHAR(50) DEFAULT 'Em Processamento',
    FOREIGN KEY (id_cliente) REFERENCES clientes(id_cliente)
);

-- 5. Tabela de Itens do Pedido
-- Tabela que conecta 'pedidos' e 'produtos'.
-- Ela registra quais produtos estão em cada pedido.
CREATE TABLE itens_pedido (
    id_item INT PRIMARY KEY AUTO_INCREMENT,
    id_pedido INT,
    id_produto INT,
    quantidade INT NOT NULL,
    preco_unitario DECIMAL(10, 2) NOT NULL,
    FOREIGN KEY (id_pedido) REFERENCES pedidos(id_pedido),
    FOREIGN KEY (id_produto) REFERENCES produtos(id_produto)
);

Inserção de Dados

Para testar as tabelas, vamos inserir alguns dados de exemplo. A ordem da inserção é importante: primeiro as tabelas sem chaves estrangeiras, depois as que dependem delas. 


-- Inserir dados nas tabelas 'categorias' e 'clientes'
INSERT INTO categorias (nome_categoria) VALUES ('Eletrônicos'), ('Livros');
INSERT INTO clientes (nome_cliente, email, endereco) VALUES
('João Silva', 'joao.silva@email.com', 'Rua A, 123'),
('Maria Souza', 'maria.souza@email.com', 'Avenida B, 456');

-- Inserir dados na tabela 'produtos' (depende de 'categorias')
-- Note que usamos os IDs das categorias que acabamos de inserir.
INSERT INTO produtos (nome_produto, descricao, preco, estoque, id_categoria) VALUES
('Smartphone X', 'Smartphone de última geração', 1500.00, 50, 1),
('Notebook Y', 'Notebook para trabalho', 2500.00, 20, 1),
('O Senhor dos Anéis', 'Livro de fantasia épica', 50.00, 100, 2);

-- Inserir dados na tabela 'pedidos' (depende de 'clientes')
INSERT INTO pedidos (id_cliente, data_pedido, status) VALUES
(1, NOW(), 'Em Processamento'),
(2, NOW(), 'Aguardando Pagamento');

-- Inserir dados na tabela 'itens_pedido' (depende de 'pedidos' e 'produtos')
-- Este pedido é do João (id_cliente = 1) e inclui 1 Smartphone e 2 livros.
INSERT INTO itens_pedido (id_pedido, id_produto, quantidade, preco_unitario) VALUES
(1, 1, 1, 1500.00),
(1, 3, 2, 50.00);

Consultas Avançadas (JOIN)

Usando o comando **JOIN**, podemos combinar dados de várias tabelas para obter informações mais completas. 


-- Consultar todos os pedidos e os dados dos clientes que os fizeram
SELECT 
    p.id_pedido,
    p.data_pedido,
    p.status,
    c.nome_cliente,
    c.email
FROM pedidos AS p
INNER JOIN clientes AS c ON p.id_cliente = c.id_cliente;

-- Consultar os detalhes de um pedido específico (ID = 1),
-- incluindo o nome do cliente, o nome do produto e a quantidade.
SELECT 
    p.id_pedido,
    c.nome_cliente,
    prod.nome_produto,
    ip.quantidade,
    ip.preco_unitario,
    (ip.quantidade * ip.preco_unitario) AS subtotal
FROM itens_pedido AS ip
INNER JOIN pedidos AS p ON ip.id_pedido = p.id_pedido
INNER JOIN clientes AS c ON p.id_cliente = c.id_cliente
INNER JOIN produtos AS prod ON ip.id_produto = prod.id_produto
WHERE p.id_pedido = 1;
